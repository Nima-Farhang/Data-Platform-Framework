# Style Guide

## SQL
- use clear CTE names
- prefer readability over dense SQL
- use descriptive column names
- avoid unnecessary nested logic

## dbt
- place models in the correct layer
- use shared macros where they reduce duplication
- use incremental models only when justified
- document important models and columns

## Naming
- use full descriptive names
- avoid abbreviations unless they are domain-standard
- keep layer naming explicit
