# Platform Principles

## 1. Build capabilities, not one-off fixes
A platform capability should be reusable across multiple dbt projects.

## 2. Remove engineer variability
Naming, schema behaviour, and common implementation patterns should be standardised where possible.

## 3. Make operations visible
Query tagging and audit metadata should make model execution easier to attribute and troubleshoot.

## 4. Keep governance close to engineering
Masking and sensitivity classification should be built into engineering workflows, not deferred to the end.

## 5. Use AI with structure
AI should work inside clear standards and repeatable task definitions.

## 6. Demonstrate application, not theory
The utilities project is only useful if the demo data product actually consumes it.
